Pourquoi une boucle fermée change tout

Dans un déploiement public, vos données traversent des serveurs tiers — anonymisées ou non. Dans la boucle fermée AInonymous, la question d'un collaborateur ne quitte jamais votre LAN ou VPN d'entreprise.

La Distributed Hash Table (DHT) Holochain tourne à l'intérieur de votre infrastructure. Les nœuds se découvrent mutuellement par votre réseau interne, et les échanges sont chiffrés de bout en bout par des clés ed25519 propres à chaque machine.

  • ✓Découverte de nœuds : via le réseau d'entreprise (pas de relais externe Nostr ou public)
  • ✓Transport : WebRTC sur LAN ou QUIC sur VPN — pas d'internet requis
  • ✓État partagé : source chain Holochain immutable, auditée, locale
  • ✓Identité : Membrane Proofs pour contrôler qui rejoint le réseau privé
  • ✓Air-gap possible : fonctionne sur réseau isolé sans accès internet

Comment une requête traverse le réseau

Du prompt de votre collaborateur à la réponse — sans jamais toucher l'extérieur.

1

Prompt utilisateur

L'utilisateur envoie une requête via son outil (VS Code, chat interne, n8n…)

2

API proxy locale

Le proxy :9337 reçoit la requête OpenAI-compat. et l'envoie au conducteur Holochain

3

Routage DHT

Le mesh consulte le registre des nœuds pour trouver les machines disponibles selon le modèle et la charge

4

Inférence distribuée

Les couches du modèle sont exécutées en pipeline sur les nœuds sélectionnés (layer-sharding)

5

Streaming tokens

Les tokens générés sont streamés vers l'API en temps réel — latence identique à du local

6

Audit chain

La tâche est chainée cryptographiquement pour traçabilité. Aucune donnée sensible stockée.

Why a closed loop changes everything

In a public deployment, your data travels through third-party servers — anonymized or not. In the AInonymous closed loop, an employee's query never leaves your corporate LAN or VPN.

The Holochain Distributed Hash Table (DHT) runs inside your infrastructure. Nodes discover each other through your internal network, and all exchanges are end-to-end encrypted with ed25519 keys unique to each machine.

  • ✓Node discovery: via the corporate network (no external Nostr or public relay)
  • ✓Transport: WebRTC over LAN or QUIC over VPN — no internet required
  • ✓Shared state: immutable, audited, local Holochain source chain
  • ✓Identity: Membrane Proofs to control who joins the private network
  • ✓Air-gap capable: works on isolated network with no internet access

No blockchain, no global consensus. Each node maintains its own immutable source chain. The DHT validates shared state without any central server.

Holochain 0.4 · Rust · WebRTC · ed25519

Layer pipeline-splitting

llama.cpp · GGUF · layer-sharding · spec-decode

Functional model

How a request traverses the network

From your employee's prompt to the response — without ever touching the outside world.

1

User prompt

The user submits a request via their tool (VS Code, internal chat, n8n…)

2

Local API proxy

The :9337 proxy receives the OpenAI-compat. request and forwards it to the Holochain conductor

3

DHT routing

The mesh consults the node registry to find available machines by model and current load

4

Distributed inference

Model layers are executed in pipeline across selected nodes (layer-sharding)

5

Token streaming

Generated tokens stream to the API in real time — latency identical to local execution

6

Audit chain

The task is cryptographically chained for traceability. No sensitive data stored.

Every layer of the system is designed to satisfy the requirements of CISOs, DPOs, and security teams.

🔑

Cryptographic identities

Every node is identified by an ed25519 key pair. No centralized authentication, no password — cryptography is the sole identity mechanism.

ed25519 · Holochain keys
🚪

Network access control

Membrane Proofs define who can join the private network. A machine without a valid proof is invisible to the mesh — even on the same LAN.

Membrane Proofs · RBAC
🔒

End-to-end encryption

All inter-node exchanges are encrypted. Data in transit is protected even on a compromised internal network.

TLS · WebRTC DTLS · E2E
📋

Immutable audit trail

Every inference task is appended to a Holochain source chain — a cryptographically linked append-only structure. Ideal for regulatory audits.

Source chain · append-only
🌐

Full network isolation

The network can run without any internet access. Supported modes: internal LAN, corporate VPN, air-gapped network. Zero dependency on external relays or DNS.

Air-gap ready · VPN · LAN
🔍

Peer validation

Holochain validation zomes allow each node to verify the conformity of other nodes' data before accepting it. No undetected malicious node.

Zome validation · warrants
