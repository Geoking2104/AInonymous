Trois couches, une seule identité

🔀

Pipeline Splitting

Un modèle est découpé entre N nœuds. Les couches s'exécutent en séquence, les activations transitent via QUIC chiffré. Aucune copie complète du modèle n'est nécessaire sur un seul nœud.

🗺️

Scheduler Topology-Aware

Le scheduler lit la topologie SD-WAN pour sélectionner le pair optimal : latence minimale, bande passante maximale, réputation Holochain, sans warrant actif.

♾️

Redondance N-of-M

PrimaryShadow, HotStandby, NofMQuorum (2/3 consensus), SpeculativeVerify — choisissez le niveau de redondance selon votre SLA métier.

Sécurité

Sécurité cryptographique à chaque niveau

Pas de confiance implicite. Chaque pair est authentifié, chaque modèle haché, chaque comportement déviant punissable par Warrant DHT.

🔑

mTLS ed25519 strict

L'AgentPubKey Holochain est réutilisé comme certificat TLS. Le PeerKeyVerifier refuse toute connexion non vérifiée — fini le dangerous_accept_any_server_cert.

📋

NodeAttestation

Chaque nœud publie dans le DHT : empreinte matérielle, benchmarks GPU, signature ed25519. Les clients refusent les pairs non attestés via require_attestation=True.

🔒

SHA-256 ModelManifest

Chaque modèle est hashé avant service. ≥ 2 pairs doivent confirmer le hash via ModelClaim. Un modèle non vérifié ne peut pas être servi.

⚖️

Warrants DHT

Preuve cryptographique de mauvaise conduite (hash incorrect, SLA violé, double-spend). Exclusion automatique du scheduling, valide 30 jours, réfutable avec preuve.

🛡️

Anti-Sybil

Score de réputation : (success×0.4) + (speed×0.3) + (age×0.2) - (warrant_penalty×0.1). PoW optionnel pour les réseaux ouverts.

🔐

Réseau privé (membrane)

PrivateNetworkProof dans le genesis — seuls les membres du consortium peuvent rejoindre le DHT. Audit log immuable via la source-chain Holochain.

🚫

Zéro pickle

Le format binaire d'activation (magic 0xA1A0C1C0 + dtype + shape + raw bytes) n'utilise jamais pickle. Pas d'exécution de code arbitraire via la désérialisation de tenseurs.

📡

DSCP 46 — Expedited Forwarding

Le trafic d'inférence est marqué DSCP 46 (EF) — traitement prioritaire sur les équipements réseau, garantie de latence sur le plan de données.

✓ Warrant-safe Audit trail complet sans serveur central

La source-chain Holochain de chaque nœud enregistre chaque attestation, chaque claim de modèle, chaque warrant publié. Immuable, non falsifiable, accessible à tout validateur du réseau.

Pipeline splitting en 5 lignes

from ainonymous_torch import PipelineSplit, LayerRange pipe = PipelineSplit( local_layers=LayerRange(model.model.layers[:18], 0, 18), remote_url="http://node-b:9337", # mTLS auto remote_layers=LayerRange(start=18, end=36), transfer_dtype=torch.float16, # -50% bande passante ) # Forward transparent — local + remote via QUIC hidden_out = pipe(hidden)

Format d'activation sécurisé

from ainonymous_torch import pack_activation, unpack_activation # Sérialisation sans pickle wire = pack_activation(hidden_state) # Magic 0xA1A0C1C0 + dtype + shape + raw bytes # Désérialisation safe tensor = unpack_activation(wire) # bfloat16 → 2× moins de bande passante # float32 → précision maximale # Supporte: f32, f16, bf16, i8, i32, i64

Wrapper transparent avec fallback local

from ainonymous_torch import DistributedInferenceModule model = DistributedInferenceModule( MyModel(), ainonymous_url="http://localhost:9337", fallback_to_local=True, # dégradation gracieuse ) out = model(x) # réseau si dispo, local sinon print(model.stats) # {"remote_calls": 42, "local_fallbacks": 1}

Use Cases

Qui utilise AInonymous ?

Des architectures sans compromis pour les contextes les plus exigeants.

Enterprise

Consortium bancaire

5 banques partagent un réseau d'inférence privé (PrivateNetworkProof). Aucune donnée client ne quitte le périmètre réglementaire. Audit trail Holochain pour conformité RGPD.

Recherche

Cluster universitaire

20 nœuds GPU sur 3 campus interconnectés via SD-WAN. Les modèles 70B s'exécutent en pipeline splitting inter-site. Chaque contribution est attestée.

Enterprise

MSP / Cloud souverain

Fournisseur de services managés proposant de l'inférence IA souveraine à ses clients. Réputation par Warrant, SLA garanti par SD-WAN, facturation par attestation.

Edge

Déploiement industriel

Usines connectées en SD-WAN. Les modèles de vision (qualité, maintenance prédictive) s'exécutent sur GPU edge avec fallback vers le cloud interne si SLA non respecté.

Recherche

Fine-tuning distribué

Les gradients transitent via le même canal QUIC sécurisé que l'inférence. La topologie SD-WAN optimise le transfer learning entre nœuds à haute bande passante.

Enterprise

Defense / Souveraineté

Réseau air-gapped avec bootstrap privé Holochain. ed25519 + mTLS sans CA externe. Les warrants remplacent le besoin d'une PKI centralisée.

Stack

Stack technique complète
